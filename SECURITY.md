# Security Policy

## Supported Versions

Security updates are handled from the repository default branch. If this project publishes releases later, supported release lines will be listed here.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately. Use GitHub private vulnerability reporting or security advisories when available. If that is unavailable, contact the maintainer directly with enough detail to reproduce or assess the issue.

Please do not open public issues for sensitive vulnerability details. The maintainer will review reports, coordinate a fix when needed, and publish remediation notes once disclosure is safe.
